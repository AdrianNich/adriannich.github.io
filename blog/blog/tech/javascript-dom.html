Javascript and it's relationship to HTML and CSS

If HTML and CSS were middle management, Javascript is the CEO. HTML and CSS work together and make sure the storefront is operating as per the Boards directive but Javascript has the overriding say as to what HTML and CSS do. Javascript can intervene directly with middle management's operations, it also has the power to introduce more customer interaction, it's head and heels above HTML and CSS in that regard. It can request information from them and use that information to run other operations and it can run those operations as many times as it needs to meet the customers needs.

The difference between accessing data from Arrays and Objects

The difference between the two is Arrays are lists, organised by their index position. For example shoppingList = ["chocolate", "bread", "milk", "cheese"] would hold "chocolate" at index position 0, "bread" at position 1, "milk" at positon 2 and so on. To access the individual items from the index we access it through the name of the array followed by the index position in square brackets. For example to access cheese we would write shoppingList[3].
Objects on the other hand are unordered lists, of which the list items can each hold a corresponding value. Each item and it's value being assigned to a key. For example if the shopping list was an object it could include the item and also the quantity of each that we require:

shoppingList = {
chocolate: 7,
bread: 2,
milk: 2,
cheese: 1
}

To access an item and it's corresponding value in an object we need to do so by using a dot notation. For example to return the amount of chocolate we would need we would write: shoppingList.chocolate

Functions and why they're useful

A function is a segment of coding which can be called to run simply by calling said function.
They are extremely useful if situations when we need to run that same segment of coding several times in different areas of the project, as it removes the need to write out that segment of code each time it is needed.
