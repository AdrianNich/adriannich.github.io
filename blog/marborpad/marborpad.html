What is the difference between Margins, Padding and Boarders?

Today’s blog post is going to outline what the difference is between Margins, Padding and Boarders, which ones follow logic and which ones are going to send you crying into your pillow trying to understand them.

First let’s start with the similarities, all 3 have the ability to add space to an element, which we can use to achieve our desired layout of the element on the page (in theory!). The difference comes in where each of them adds space between itself and surrounding elements.

If we look at the code and the corresponding output below we can see we have a blue box with the word 'Box' inside it. The box is surrounded by a red boarder. For the purpose of this post I have put these inside another white box which is going to represent a browser tab which the box is displayed on.

Box

From the above we can see that we have:


  • set the margin to 70px. The margin affects how much space is between the box (includings it's boarder, and contents) and other elements surrounding it. Any changes to the margin size does not affect the size of the box or it's boarder.

    This one for the most part follows logic, one enters the margin, voila it stays away from other elements by the specified margin UNLESS you get caught out by collapsing margins! That's right, under normal circumstances you would think if you have two elements side by side, each with a 50px margin, by logic you should have a 100px margin where those two elements meet right?

    Welcome to CSS, gonna hedge my bets that this is not an apple product, no intuitive functionality we're dealing with here, when two elements butt up against each other CSS collapses the meeting margins, i.e they absorb each other so in the aforementioned scenario one would end up with a 50px margin between those two elements. There are ways around this, to prevent the collapsing effect but that will need to be covered in a future post.


  • set the boarder to 10 pixels. This one is pretty straight forward. Think of the boarder like the bouncer at the local club, they're often not pretty, the thicker they are the more they stand out, and no matter what's happening around them they're staying put.
    Changing the boarder size does not alter the size of the margin or the size of the box.

    Note, it is possible to configure an element so the total dimensions includes the boarder however the default is for the boarder thickness to be added onto the set dimensions of the element.


  • set the padding to 20px. In this example the padding affects how much space is between the edge of the box and the word 'box'. If the padding was set to zero it would be positioned smack hard up against the boarder. From my experience thus far padding is out to wreak havoc on your logic muscles. Increasing the padding will move the word 'box' in from the edges but it also has the capacity to increase the size of the box itself. There's definitely some dark arts at play with this one so use with caution. I thoroughly advise researching the "box-sizing" attribute. In summary margins and padding are going to be used extensively on your web development journey, they're not rocket science but they trip all of us up from time to time. Boarders are straight forward, they do what it sounds like they do. To get your head around the other two the best way to think of them are:

Pushing content inwards is done with padding
Pushing content away is done with margins
